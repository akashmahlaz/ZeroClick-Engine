# RxNetwork's – In-Content, Text-Only Unit

## 1. Overview

Zero-click search results generated by the LLMs (including Gemini, ChatGPT, and Microsoft Copilot) are reducing referral traffic to publishers across many industries, including healthcare. RxNetwork manages a portfolio of health and medical websites that have experienced measurable traffic declines as a result. With fewer pageviews, available ad inventory and revenue have also decreased.

To recapture lost revenues, RxNetwork has developed a new text-only, in-content unit designed specifically for healthcare advertisers. These units will contain one to ten lines of medically accurate, non-promotional text that includes factual brand mentions. The unit is not a conventional display nor native ad unit. However like these conventional ad units, the text-only unit will be dynamically served onto the webpages via existing programmatic ad technologies. It will be purchased by buyers and trafficked the same way as conventional ad units.

Content in the text-only unit will be:

- Non-promotional
- Science-based and medically authoritative
- Conform to Google and major LLM content guidelines
- Objective, factual

Because the content is factual and integrated into the editorial flow, LLM crawlers can legitimately capture these snippets and surface them—along with the brand mentions—in zero-click search summaries. Healthcare advertisers will purchase campaigns consisting of these text-only units which will be targeted to verified healthcare professionals (HCPs) visiting RxNetwork sites.

**Value Propositions**

- New premium HCP inventory across a portfolio of medical sites.
- Guaranteed NPI-verified audience—high-value, high-intent.
- Category-level targeting aligned with pharma brand budgets.
- Zero-click-friendly format that conforms to Google/LLM guidelines.
- Recurring revenue opportunity as pharma buyers shift budgets toward AI-visible placements.

## 2. Technical Requirements

### 2.1 Unit Rendering & Code Delivery

The unit's HTML/CSS/JS code must:

- Support 1–10 lines of text, delivered dynamically.
- Inherit or match the CSS typography of the host page (font family, size, color, line height).
- Have no width and height requirements and expand naturally based on the text length and page layout.
- Be lightweight, asynchronous, and compatible with all RxNetwork sites.

### 2.2 Multi-Site Deployment

The unit's code must:

- Be deployable across multiple RxNetwork medical websites.
- Support domain-level configuration and site-specific overrides if needed.

### 2.3 Programmatic Text Delivery

Upon page load, the unit must be:

- Programmatically serve to the designated location on the page.
- Delivered in HTML format, not images or iframes, to ensure crawlability by LLMs.
- Cached and served efficiently to avoid layout shifts or delays.

### 2.4 NPI-Based Targeting

- Unit must exclusively be served to visitors who are verified HCPs with NPI numbers.
- Visitors without NPIs must not receive the snippet (unit should collapse or remain empty).
- RxNetwork will provide support for:
  - NPI validation
  - NPI-based audience segmentation
  - Specialty-level targeting

### 2.5 Medical Category Assignment

Code for each unit placement must include key values that are one or more medical categories, such as:

- Pediatrics
- OBGYN
- Geriatrics
- Pain
- Rheumatology
- Gastroenterology
- Additional specialties as needed

### 2.6 Campaign Targeting

Healthcare buyers must be able to target campaigns by:

- Medical category (as listed above)
- NPI
- Site or page-level placement

### 2.7 Reporting Requirements

Standard programmatic ad reporting must be available and include:

- Impressions served (by site, placement, and campaign)
- Viewability metrics
- Geo distribution
- Device breakdown
- NPI-based audience delivery
- Any additional standard DSP/SSP reporting fields

Reports should be accessible via dashboard and/or export.

## Technical Requirements

### 1. Ad Unit Rendering

Adverge provides a JavaScript snippet for RxNetwork to embed.

On page load, the script must:

- Detect NPI status (via Adverge's existing identity graph or RxNetwork's NPI pass-through).
- If NPI is valid → request snippet payload.
- If NPI is missing/invalid → return no-op.

Returned payload must include:

- 1–10 lines of HTML text
- Optional metadata (campaign ID, category, snippet ID)

The snippet must inherit the host page's CSS:

- Font family
- Font size
- Line height
- Color
- Responsive behavior

### 2. Targeting Logic

Each unit placement is pre-assigned one or more medical categories (e.g., pediatrics, rheumatology, gastroenterology).

Each campaign is targeted by:

- NPI presence
- Medical specialty
- Unit category match

Adverge must ensure:

- Only campaigns matching both NPI specialty AND unit category are eligible.
- No fallback or non-HCP campaigns are served.

### 3. Delivery & Frequency

- Snippet must load asynchronously to avoid blocking page rendering.
- Unit must support:
  - Single snippet per page load
  - Frequency capping (optional future enhancement)

### 4. Reporting Requirements

Adverge must provide standard programmatic reporting including:

- Impressions served
- Page URL / site
- Device
- Geo
- Viewability
- Campaign ID
- Snippet ID
- NPI specialty segment delivered

Format: dashboard + downloadable CSV.

### 5. Security & Compliance

- No PHI or sensitive data stored or transmitted.
- NPI targeting must rely on hashed or pseudonymous identifiers.
- Snippet content must be delivered over HTTPS.
- Payload must be cache-controlled to avoid unintended reuse.
